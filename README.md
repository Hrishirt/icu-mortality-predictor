# ICU In-Hospital Mortality Prediction

Clinical outcome prediction system built on the [PhysioNet Challenge 2012](https://physionet.org/content/challenge-2012/1.0.0/) ICU mortality dataset. The pipeline loads patient vitals and lab values, trains and compares a Gradient Boosting classifier and a PyTorch MLP, explains predictions with SHAP, and serves risk scores through a FastAPI endpoint with LLM-generated plain-English summaries. All predictions are logged to PostgreSQL for auditability.

## Features

- **Data preprocessing** — Aggregates time-series vitals/labs per patient; handles missing values with median imputation
- **Dual models** — scikit-learn Gradient Boosting (default) and a simple PyTorch MLP
- **Evaluation** — AUC-ROC, precision, recall, F1, and accuracy on a held-out test set
- **SHAP explainability** — Per-prediction feature contributions via `TreeExplainer`
- **FastAPI service** — `POST /predict` returns mortality risk, top SHAP features, and an explanation
- **LLM summaries** — LangChain + OpenAI for clinician-friendly explanations (template fallback without API key)
- **Audit logging** — PostgreSQL stores every prediction, input features, SHAP values, and explanation text
- **Docker** — `docker-compose` runs the API and PostgreSQL together

## Model performance

Evaluated on 800 held-out patients (20% test split):

| Model | AUC-ROC | Precision | Recall | F1 |
|-------|---------|-----------|--------|-----|
| Gradient Boosting | 0.873 | 0.657 | 0.207 | 0.315 |
| MLP (PyTorch) | 0.804 | 0.451 | 0.288 | 0.352 |

Gradient Boosting is the default production model (`DEFAULT_MODEL=gradient_boosting`).

## Quick start

### 1. Clone and install

```bash
git clone <your-repo-url>
cd roche
pip install -r requirements.txt
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env` as needed:

```env
DATABASE_URL=postgresql://mortality_user:mortality_pass@localhost:5432/mortality_db
OPENAI_API_KEY=your-openai-api-key-here   # optional — enables LLM explanations
OPENAI_MODEL=gpt-4o-mini
MODEL_DIR=artifacts
DATA_DIR=data/physionet
```

### 3. Download the dataset

**Linux / macOS:**

```bash
wget -r -N -c -np https://physionet.org/files/challenge-2012/1.0.0/ -P data/physionet
```

**Windows (PowerShell):**

```powershell
.\scripts\download_data.ps1
```

This downloads `Outcomes-a.txt` and `set-a/` (4,000 labeled ICU stays with vitals and lab time series).

### 4. Train models

```bash
python scripts/train_models.py
```

Outputs saved to `artifacts/`:

- `gradient_boosting.joblib` — trained GBM pipeline
- `mlp.joblib` / `mlp_torch.pt` — trained MLP
- `feature_names.json` — 257 feature names
- `metrics.json` — evaluation results
- `shap_background.npy` — background data for SHAP

Training takes ~2 minutes on a typical laptop.

### 5. Run the API

**Local:**

```bash
# From project root
export PYTHONPATH=.          # Linux/macOS
# $env:PYTHONPATH="."        # Windows PowerShell

uvicorn src.api.main:app --host 0.0.0.0 --port 8000
```

**Docker (API + PostgreSQL):**

```bash
docker compose up --build
```

Open interactive docs at [http://localhost:8000/docs](http://localhost:8000/docs).

## API reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/health` | Service status and model load state |
| `GET` | `/metrics` | Model evaluation metrics |
| `GET` | `/features` | List of required input feature names |
| `POST` | `/predict` | Mortality risk score + SHAP + explanation |

### Example prediction

Build a payload from a patient record:

```bash
python scripts/sample_request.py 132551 > payload.json
```

Send the request:

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d @payload.json
```

Example response:

```json
{
  "patient_id": "132551",
  "model_name": "gradient_boosting",
  "mortality_risk_score": 0.349,
  "mortality_risk_percent": 34.91,
  "top_shap_features": [
    {"feature": "PaCO2_mean", "shap_value": 0.382},
    {"feature": "Urine_mean", "shap_value": 0.362},
    {"feature": "BUN_min", "shap_value": 0.267}
  ],
  "explanation": "The model estimates a 34.9% in-hospital mortality risk...",
  "prediction_id": 1
}
```

### Request schema

```json
{
  "patient_id": "optional-string",
  "features": {
    "Age": 54.0,
    "HR_mean": 73.0,
    "Temp_mean": 35.1
  }
}
```

Feature keys must match the names in `artifacts/feature_names.json`. Use `GET /features` or `scripts/sample_request.py` to see the full list. Missing features are median-imputed at inference time.

## How it works

### Preprocessing

Each patient record is a CSV time series of vitals and labs. The pipeline:

1. Extracts static fields (Age, Gender, Weight, ICUType, etc.)
2. Aggregates each time-varying parameter into 7 statistics: mean, median, min, max, std, last, count
3. Replaces sentinel values (`-1`) with NaN
4. Applies median imputation during model training and inference

### Models

- **Gradient Boosting** — `sklearn.ensemble.GradientBoostingClassifier` inside a `Pipeline` with `SimpleImputer(strategy="median")`
- **MLP** — 3-layer feedforward network in PyTorch (64 → 32 → 1) with dropout, trained with Adam and BCE loss

### Explainability

SHAP `TreeExplainer` computes per-feature contributions for each GBM prediction. Positive SHAP values push risk up; negative values push it down.

### LLM explanations

When `OPENAI_API_KEY` is set, LangChain sends the risk score and top SHAP features to an OpenAI model and returns a 2–3 sentence clinical summary. Without a key, a deterministic template-based fallback is used.

### Audit logging

Every `/predict` call is stored in PostgreSQL:

```sql
SELECT id, created_at, patient_id, risk_score, explanation
FROM prediction_logs
ORDER BY created_at DESC;
```
